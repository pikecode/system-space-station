## 一、技术栈

| 层 | 选型 | 说明 |
|---|---|---|
| 前端 | React 18 + TypeScript | 类型安全，组件生态成熟 |
| 前端 UI | Ant Design Pro 5.x | 企业后台标准组件库，内置 RBAC 布局 |
| 前端路由 | React Router v6 | 嵌套路由支持角色布局隔离 |
| 前端状态 | Zustand | 轻量，适合中小型后台 |
| 前端请求 | TanStack Query + Axios | 缓存、loading、错误处理统一管理 |
| 后端框架 | NestJS + TypeScript | 模块化，依赖注入，适合 RBAC 权限体系 |
| 数据库 | PostgreSQL | 事务支持强，分成计算原子性有保障 |
| ORM | Prisma | 类型安全，迁移管理清晰 |
| 认证 | JWT（Access Token） | 无状态，前后端分离友好 |
| 小程序端 | Taro 4.x + React + TypeScript | 与 Web 同语言，共享 packages/shared 类型，支持微信小程序 / H5 双端 |
| 工程管理 | pnpm monorepo | 前端、小程序、后端共享 TypeScript 类型定义 |

## 四、数据模型

### 4.1 用户与组织

```prisma
model Department {
  id        String   @id @default(cuid())
  name      String
  headId    String?          // 部门负责人 userId
  status    DeptStatus @default(ACTIVE)
  createdAt DateTime @default(now())

  head      User?    @relation("DeptHead", fields: [headId], references: [id])
  members   User[]   @relation("DeptMembers")
  customers Customer[]
}

model User {
  id           String   @id @default(cuid())
  name         String
  phone        String   @unique
  passwordHash String
  role         UserRole             // ADMIN | HEAD | MEMBER
  departmentId String?
  status       UserStatus @default(ACTIVE)
  createdAt    DateTime @default(now())

  department   Department? @relation("DeptMembers", fields: [departmentId], references: [id])
  headOf       Department? @relation("DeptHead")
  customers    Customer[]
}

enum UserRole   { ADMIN HEAD MEMBER }
enum UserStatus { ACTIVE INACTIVE }
enum DeptStatus { ACTIVE INACTIVE }
```

### 4.2 客户

```prisma
model Customer {
  id           String   @id @default(cuid())
  name         String
  phone        String
  idCard       String?
  source       String?          // 来源渠道
  notes        String?
  assignedTo   String           // 维护人 userId（1:1）
  departmentId String
  status       CustomerStatus @default(ACTIVE)
  createdAt    DateTime @default(now())
  updatedAt    DateTime @updatedAt

  assignedUser User        @relation(fields: [assignedTo], references: [id])
  department   Department  @relation(fields: [departmentId], references: [id])
  memberships  Membership[]
}

enum CustomerStatus { ACTIVE INACTIVE }
```

### 4.3 会员

```prisma
model Membership {
  id           String           @id @default(cuid())
  customerId   String
  fee          Decimal          @db.Decimal(10, 2)
  startDate    DateTime
  endDate      DateTime
  status       MembershipStatus @default(PENDING)
  submittedBy  String           // 提交人 userId
  reviewedBy   String?          // 审批人 userId
  reviewNote   String?
  createdAt    DateTime         @default(now())
  updatedAt    DateTime         @updatedAt

  customer     Customer  @relation(fields: [customerId], references: [id])
  commissions  CommissionRecord[]
}

enum MembershipStatus {
  PENDING       // 待审核
  APPROVED      // 已审批（有效）
  REJECTED      // 已拒绝
  EXPIRED       // 已到期
  REFUND_PENDING // 退款申请中
  REFUNDED      // 已退款
}
```

### 4.4 分成

```prisma
model CommissionConfig {
  id              String   @id @default(cuid())
  memberRatio     Decimal  @db.Decimal(5, 2)    // 维护人比例（快照时使用）
  headRatio       Decimal  @db.Decimal(5, 2)    // 负责人比例
  companyRatio    Decimal  @db.Decimal(5, 2)    // 公司比例
  settlementDays  Int      @default(15)
  effectiveFrom   DateTime
  createdAt       DateTime @default(now())
}

model CommissionRecord {
  id           String           @id @default(cuid())
  membershipId String
  receiverId   String           // 收款方 userId（公司层填公司虚拟 ID）
  receiverRole CommissionRole   // MEMBER | HEAD | COMPANY
  amount       Decimal          @db.Decimal(10, 2)
  ratio        Decimal          @db.Decimal(5, 2)  // 比例快照
  status       CommissionStatus @default(PENDING)
  periodId     String?
  createdAt    DateTime         @default(now())

  membership   Membership        @relation(fields: [membershipId], references: [id])
  period       SettlementPeriod? @relation(fields: [periodId], references: [id])
}

model SettlementPeriod {
  id          String           @id @default(cuid())
  startDate   DateTime
  endDate     DateTime
  status      PeriodStatus     @default(OPEN)
  settledBy   String?
  settledAt   DateTime?
  createdAt   DateTime         @default(now())

  records     CommissionRecord[]
}

enum CommissionRole   { MEMBER HEAD COMPANY }
enum CommissionStatus { PENDING PENDING_PAYMENT SETTLED REVERSED }
enum PeriodStatus     { OPEN PENDING_PAYMENT SETTLED }
```

## 五、权限设计（RBAC）

```
ADMIN  → 所有接口
HEAD   → 本部门客户读写 + 审批接口 + 本部门报表
MEMBER → 本人名下客户读写 + 提交会员申请 + 个人分成查看
```

实现方式：NestJS `@Roles()` 装饰器 + `RolesGuard`，结合 JWT payload 中的 `role` 和 `departmentId` 做数据行级过滤。

## 六、核心 API 设计

### 认证
```
POST /auth/login          → { token }
GET  /auth/me             → 当前用户信息
```

### 部门
```
GET    /departments        → 部门列表（ADMIN）
POST   /departments        → 创建部门（ADMIN）
PATCH  /departments/:id    → 编辑部门（ADMIN）
DELETE /departments/:id    → 停用部门（ADMIN）
```

### 用户
```
GET    /users              → 用户列表（ADMIN，可按部门筛选）
POST   /users              → 创建用户（ADMIN）
PATCH  /users/:id          → 编辑用户（ADMIN）
PATCH  /users/:id/status   → 启用/停用（ADMIN）
```

### 客户
```
GET    /customers          → 客户列表（按角色过滤数据范围）
POST   /customers          → 创建客户（MEMBER/HEAD）
GET    /customers/:id      → 客户详情
PATCH  /customers/:id      → 编辑客户
PATCH  /customers/:id/transfer → 转移维护人（HEAD）
```

### 会员
```
POST   /memberships                    → 提交会员申请（MEMBER）
GET    /memberships/pending            → 待审批列表（HEAD）
PATCH  /memberships/:id/approve        → 审批通过（HEAD）
PATCH  /memberships/:id/reject         → 审批拒绝（HEAD）
POST   /memberships/:id/refund         → 申请退款
```

### 分成 & 结算
```
GET  /commissions/my                   → 个人分成明细（MEMBER）
GET  /commissions/department           → 部门分成汇总（HEAD）
GET  /commissions/overview             → 全公司汇总（ADMIN）
POST /commissions/settle/:periodId     → 财务确认结算（ADMIN）
```

### 配置
```
GET   /config              → 当前配置
PATCH /config              → 更新分成比例和结算周期（ADMIN）
```

## 七、小程序端（Taro）

### 功能定位

小程序面向**一线使用人员**（成员 + 负责人），提供移动端核心操作。系统管理员的组织管理和全局配置仅在 Web 后台操作。

### 小程序页面结构

```
tabBar
├── 客户          /pages/customers/index     我的客户列表
├── 待办          /pages/approvals/index     待审批会员申请（负责人可见）
└── 我的          /pages/profile/index       个人信息 + 分成明细

页面详情
├── /pages/customers/detail     客户详情 + 会员记录
├── /pages/memberships/create   提交会员申请
├── /pages/memberships/detail   会员申请详情（含审批操作）
└── /pages/commissions/list     分成明细列表
```

### 与 Web 后台的差异

| 功能 | Web 后台 | 小程序 |
|------|---------|--------|
| 部门/用户管理 | ✅ | ✗（管理员专属） |
| 分成比例配置 | ✅ | ✗ |
| 新增客户 | ✅ | ✅ |
| 提交会员申请 | ✅ | ✅ |
| 审批会员申请 | ✅ | ✅（负责人） |
| 个人分成查看 | ✅ | ✅ |
| 全公司分成总览 | ✅ | ✗ |
| 财务结算操作 | ✅ | ✗ |

### 复用策略

- `packages/shared` 类型三端共用（web / miniapp / api）
- 小程序 `services/` 调用与 web 完全相同的 REST API 端点
- JWT 认证逻辑一致，token 存 `wx.setStorageSync`
