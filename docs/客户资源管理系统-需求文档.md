# 客户资源管理系统 — 产品需求文档

**版本：** v1.0  
**日期：** 2026-07-17  
**状态：** 待 Review

---

## 一、背景与目标

公司下设多个部门，每个部门有负责人和若干成员。各部门人员需要维护客户信息，并为客户办理入会（收取会员费）。系统需要根据会员费自动计算并记录三层费用分成：维护人员、部门负责人、公司。

**核心目标：**
- 管理客户信息及其维护归属
- 流转会员申请的审批
- 自动计算分成，按周期汇总供财务对账

---

## 二、系统角色与权限

| 角色 | 数据可见范围 | 核心操作 |
|------|------------|---------|
| 系统管理员 | 全公司 | 管理部门、人员；配置分成比例和结算周期 |
| 部门负责人 | 本部门 | 审批会员申请；查看本部门客户和分成 |
| 部门成员 | 自己维护的客户 | 录入客户；提交会员申请；查看个人分成 |

---

## 三、组织管理

- 支持部门的创建、编辑、停用
- 每个部门指定唯一负责人
- 成员只能归属一个部门
- 系统管理员可调整成员所属部门（调整后历史分成不追溯）

---

## 四、客户管理

- 成员录入客户，客户录入时绑定到当前维护人（1 客户 : 1 维护人）
- 成员只能编辑自己名下的客户；负责人可编辑本部门所有客户
- 负责人可将客户的维护人变更给本部门其他成员，历史分成不追溯
- 客户列表权限隔离：成员只看自己名下，负责人看全部门，管理员看全公司

---

## 五、会员管理

### 5.1 会员状态机

```
                    ┌─── 审批拒绝 ──→ 已拒绝（可重新提交）
待审核 ──→ 审批中 ──┤
                    └─── 审批通过 ──→ 有效 ──→ 到期
                                        │
                                        └──→ 退款申请 ──→ 已退款
```

### 5.2 流程说明

1. **提交申请**：成员为名下客户填写会员费金额、有效期，提交后状态为"待审核"，不触发分成
2. **负责人审批**：
   - 审批通过 → 状态变"有效"，触发分成计算
   - 审批拒绝 → 填写拒绝原因，成员可修改后重新提交
3. **到期**：有效期结束，状态自动变"到期"，维护权不变，不产生新分成
4. **退款**：发起退款申请，审核通过后生成等额负向冲账记录，下一结算周期抵扣

---

## 六、费用分成

### 6.1 分成计算规则

会员审批通过时，系统自动生成三条分成记录：

```
维护人员分成 = 会员费 × 维护人比例
部门负责人分成 = 会员费 × 负责人比例
公司分成     = 会员费 × 公司比例

约束：三项比例之和 = 100%（系统强制校验）
```

> **特殊情况**：若负责人本人是该客户的维护人，负责人层不再单独计算，维护人员直接取维护人比例，负责人比例部分归入公司分成（或按后台配置处理，待确认）。

### 6.2 分成结算流程

```
会员审批通过
    ↓
生成分成记录（状态：待结算）
    ↓
到达结算周期截止日
    ↓
汇总本周期所有"待结算"记录 → 状态变"待出账"
    ↓
财务手动核对并确认
    ↓
标记为"已结算"，记录操作人和操作时间
```

### 6.3 分成状态

| 状态 | 说明 |
|------|------|
| 待结算 | 会员有效，等待下一结算周期 |
| 待出账 | 结算周期到，汇总完毕，等待财务确认 |
| 已结算 | 财务手动确认完成 |
| 已冲账 | 退款产生的负向记录 |

### 6.4 结算周期规则

- 默认每 15 天结算一次，后台可配置（1～365 天）
- 结算周期修改仅对下一周期生效，当前进行中的周期不受影响
- 分成比例修改仅对修改后新产生的会员记录生效，历史记录保留修改时的比例快照

---

## 七、后台配置（仅系统管理员）

| 配置项 | 说明 | 约束 |
|--------|------|------|
| 维护人员分成比例 | 百分比 | 与其他两项之和必须 = 100% |
| 部门负责人分成比例 | 百分比 | 同上 |
| 公司分成比例 | 百分比 | 同上 |
| 结算周期天数 | 整数天 | 1～365，默认 15 |

---

## 八、报表与查询

| 角色 | 可查看内容 |
|------|----------|
| 成员 | 个人分成明细：客户名、会员费、本人分成金额、分成状态、产生时间 |
| 负责人 | 本部门汇总：各成员分成明细、本人负责人分成、部门周期汇总 |
| 系统管理员 | 全公司汇总：各部门汇总、公司分成汇总、结算历史记录 |

---

## 九、边界规则与异常场景

| 场景 | 处理方式 |
|------|---------|
| 负责人直接维护客户 | 负责人层不单独计分成，只取维护人比例（待确认具体规则） |
| 维护人变更 | 负责人操作，历史分成不追溯，新会员按新维护人计算 |
| 会员退款 | 生成负向冲账记录，下一结算周期抵扣，不直接回收已结算金额 |
| 会员到期未续费 | 状态变"到期"，维护权保留，不产生新分成 |
| 比例配置修改 | 仅新记录生效，已有分成记录快照保留修改前比例 |
| 结算周期内有冲账 | 冲账记录与正向记录在同一周期汇总，可出现负数净额 |

---

## 十、一期范围边界（不做）

- 不对接第三方支付，仅做账务记录
- 不支持一个客户同时由多人维护
- 不支持按部门或会员等级差异化分成比例（全公司统一一套）
- 不做税务处理和财务凭证导出
- 不做跨部门客户共享
